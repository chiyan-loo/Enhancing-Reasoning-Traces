## Overview

This repository provides tools to generate long, high-quality reasoning traces for mathematical problem-solving. Instead of relying on a larger teacher model, we use an iterative **"Wait-Until-Correct" (WUC)** generation process. If a model fails to reach the correct answer, we force it to continue thinking using budget tokens like "Wait" or "Alternatively". Preliminary results show that models trained on reasoning traces generated by this method significantly outperform models trained on standard reasoning traces (small scale experiments with Qwen2.5-3B, QLoRA, and ~400 training samples).

## Structure

- `reasoning/`: Synthetic reasoning generation scripts using iterative budget forcing.
- `train/`: Training scripts for Supervised Fine-Tuning (SFT) with LoRA.
- `eval/`: Scripts for evaluating models on benchmarks like MATH-500.

## Reasoning Trace Generation

Generate reasoning traces using an iterative loop that continues until the model reaches a correct answer or hits a maximum step limit.

### Key Features
- **Iterative Budget Forcing**: Appends "Wait" or "Alternatively" when the model's intermediate answer is incorrect.
- **Decaying Over-Generation**: Optionally continues generating even after a correct answer is found to encourage longer, more complex reasoning chains.
- **Robust Math Verification**: Uses `math_verify` to accurately grade LaTeX and mathematical expressions.
- **vLLM Integration**: Fast inference and batch processing.

### Example Command

```bash
python3 reasoning/generate_traces.py \
    --model Qwen/Qwen2.5-7B-Instruct-AWQ \
    --dataset nlile/hendrycks-MATH-benchmark \
    --split train \
    --mode budget \
    --max_steps 5 \
    --over_gen_prob 0.8 \
    --over_gen_exponent 3.0 \
    --temperature 0.8 \
    --num_samples 1000 \
    --output_file MATH_traces_bf.jsonl
```

### Configuration Parameters

| Argument | Description | Default |
| :--- | :--- | :--- |
| `--mode` | `budget` (repeats "Wait") or `alternating` (alternates "Wait"/"Alternatively"). | `budget` |
| `--max_steps` | Maximum number of rethink iterations. | `10` |
| `--over_gen_prob` | Base probability to continue thinking even if the answer is correct. | `0.2` |
| `--over_gen_exponent`| How fast the over-gen probability decays over steps (higher = faster decay). | `1.0` |
| `--model` | HuggingFace model path or name (optimized for vLLM). | `Qwen/Qwen2.5-7B-Instruct-AWQ` |

## Workflow

### 1. Generate Training Data
Use the reasoning script above to create a dataset of correct reasoning traces (`traces_correct.jsonl`).

### 2. Fine-Tune with LoRA
Train a smaller student model on the generated high-quality traces.

```bash
python train/lora.py \
    --model_name_or_path "Qwen/Qwen2.5-3B" \
    --dataset_name_or_path "reasoning/MATH_traces_bf_correct.jsonl" \
    --dataset_split "train" \
    --output_dir "./output/Qwen2.5-3B-SFT" \
    --per_device_train_batch_size 1 \
    --gradient_accumulation_steps 16 \
    --system_prompt "Reason step by step, and put your final answer within \\boxed{}."
```

### 3. Merge and Evaluate
Merge the adapters and evaluate the resulting model.

```bash
# Merge
python train/merge_lora.py \
    --base_model_name_or_path "Qwen/Qwen2.5-3B" \
    --adapter_path "./output/Qwen2.5-3B-SFT" \
    --output_dir "./output/Qwen2.5-3B-SFT-merged"

# Evaluate
lm-eval run \
    --model vllm \
    --model_args pretrained=./output/Qwen2.5-3B-SFT-merged \
    --tasks minerva_math500 \
    --apply_chat_template \
    --output_path ./results
```
